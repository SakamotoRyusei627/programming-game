## 基本方針

- E2E は主要フローに絞り、単体テストで代替できる詳細までは持ち込まない
- 環境依存を最小化する
- 失敗時に切り分けできる粒度を意識する

## 実装ルール

- E2E フレームワーク: 要確認

- ローカル/CI 実行条件: 要確認
- スクリーンショットや録画方針: 要確認
- 変更時は待機条件と安定性を確認する

## よくある変更パターン

### パターン1: 主要導線テスト追加

- 手順:
  1. 利用者フローを短く安定した形で切る
  2. 前提データと後始末を整理する

### パターン2: 不安定テスト修正

- 手順:
  1. 失敗要因が同期不良かデータ依存かを切り分ける
  2. 待機条件やテストデータ準備を見直す
