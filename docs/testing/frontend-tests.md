# Frontend Tests

## AI向け要約

- フロントエンドの単体・統合・UIテスト方針を整理する文書です。
- 実装とテスト基盤は未確認です。
- 画面変更やコンポーネント変更時の検証観点を揃えるために参照します。

## このドキュメントを読むべきとき

- UIテストを追加するとき
- コンポーネント変更の影響範囲を確認したいとき
- モック方針や描画確認方法を確認したいとき

## 基本方針

- UIテストは利用者から見える振る舞いを優先する
- 実装詳細への過剰な依存を避ける
- 通信や状態管理のモック境界を明確にする

## 実装ルール

- テストランナー: 要確認
- レンダリングヘルパー: 要確認
- モック方式: 要確認
- スナップショット利用方針: 要確認
- 変更時はアクセシビリティ観点も確認する

## よくある変更パターン

### パターン1: コンポーネントテスト追加

- 手順:
  1. 利用者視点の期待値を整理する
  2. 必要最小限のモックでテストを書く

### パターン2: フォームテスト追加

- 手順:
  1. 入力とエラー表示の流れを確認する
  2. 送信成功・失敗の両方を検証する
