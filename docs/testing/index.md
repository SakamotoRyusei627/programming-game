# Testing Index

## AI向け要約

- このカテゴリは、フロントエンド、バックエンド、E2E のテスト方針を扱います。
- 現時点ではテスト基盤やコマンドは未確認です。
- テスト追加・修正のときに最初にここを読み、対象に応じた詳細だけ確認します。
- 実装が追加されたら、実行コマンドと観点を具体化してください。
- 共通の実装フローは TDD を前提とし、実行コマンドは原則 `Makefile` 経由で統一します。

## このカテゴリを読むべきとき

- テストを追加するとき
- 失敗テストの原因を切り分けるとき
- どの粒度で検証すべきか確認したいとき

## 収録ドキュメント

| ファイル | 役割 | いつ読むか |
|---|---|---|
| [frontend-tests.md](./frontend-tests.md) | UI/フロントテスト | UI変更時 |
| [backend-tests.md](./backend-tests.md) | API/業務ロジックテスト | サーバ変更時 |
| [e2e-tests.md](./e2e-tests.md) | E2E テスト | 横断フロー変更時 |

## 現在の前提

- 最小構成として `node:test` のスモークテストを `tests` 配下で運用しています。
- アプリレベルの検証は `next build` を利用します。
- 変更前後の確認は、原則 `make check` を入口にします。
