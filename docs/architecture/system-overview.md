# System Overview

## AI向け要約

- システム全体の責務分割、主要レイヤ、境界を整理するための文書です。
- 現時点では実装が未確認のため、構成の断定はしていません。
- アプリケーション全体の説明が必要な変更時だけ読みます。
- 技術選定や構成が確定したら、この文書を修正して入口情報の精度を上げます。

## このドキュメントを読むべきとき

- 全体像を把握してから実装方針を決めたいとき
- フロントエンドとバックエンドの責務境界を確認したいとき
- 新しいサブシステムを追加するとき

## 関連ファイル

| 種別 | パス | 説明 |
|---|---|---|
| 実装 | 未確認 | アプリ全体の入口ファイルは要確認 |
| テスト | 未確認 | システム全体を検証するテストは要確認 |
| 関連docs | [docs/architecture/directory-structure.md](./directory-structure.md) | 配置ルールの確認 |

## 基本方針

- 全体像は、責務分離と変更影響の見通しを優先して記述する
- コード断片の説明ではなく、レイヤ間の関係と制約を書く
- 未確定の構成は断定せず、`要確認` として残す

## 実装ルール

- レイヤ構成: 要確認
- クライアントとサーバの境界: 要確認
- バッチや非同期処理の有無: 要確認
- 変更時は関連カテゴリの `index.md` も確認する

## よくある変更パターン

### パターン1: 新しい機能領域を追加する

- 手順:
  1. 既存レイヤに置くべきか `要確認`
  2. 依存先と責務境界への影響を確認する

### パターン2: システム構成を変更する

- 手順:
  1. 実装とデプロイへの影響範囲を整理する
  2. 変更後に関連 `docs` を更新する
