# External Services

## AI向け要約

- この文書は、外部API、SaaS、認証基盤、通知基盤などの外部依存を整理する場所です。
- 連携先が未確認のため、現時点では確認観点のみを置いています。
- 外部依存の追加・変更・障害調査のときに参照します。
- 認証情報そのものは書かず、扱い方と参照先だけ記載します。

## このドキュメントを読むべきとき

- 外部サービス連携を追加するとき
- 接続先変更の影響を確認するとき
- 障害切り分けで依存先を確認したいとき

## 関連ファイル

| 種別 | パス | 説明 |
|---|---|---|
| 実装 | 未確認 | 外部連携コードは要確認 |
| テスト | 未確認 | モックや統合テストは要確認 |
| 関連docs | [docs/security/secrets.md](../security/secrets.md) | 秘密情報の扱い |

## 基本方針

- 接続先の責務、認証方式、失敗時の扱いを中心に整理する
- 秘密情報の値は記載しない
- 依存先追加時は運用・監視・デプロイ影響も合わせて確認する

## 実装ルール

- サービス名一覧: 要確認
- 認証方式: 要確認
- タイムアウトやリトライ方針: 要確認
- ローカル開発時の代替手段: 要確認
- 関連する環境変数名は必要なら記載し、値は記載しない

## よくある変更パターン

### パターン1: 連携先を追加する

- 手順:
  1. 認証方式と失敗時の扱いを整理する
  2. 監視と秘密情報管理の更新要否を確認する

### パターン2: 接続設定を変更する

- 手順:
  1. 環境差分の有無を確認する
  2. 影響を受ける `docs` とテストを更新する
