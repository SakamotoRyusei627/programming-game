# Directory Structure

## AI向け要約

- この文書は、ディレクトリごとの責務と配置方針を整理する場所です。
- 現時点では実装ディレクトリが未確認のため、確定情報はありません。
- 新規ファイルの配置先を判断するときに参照します。
- 実際の構成が固まったら、ここを入口として更新します。

## このドキュメントを読むべきとき

- 新しいファイルの配置先を決めるとき
- 命名規則や責務分離を確認したいとき
- ディレクトリ再編の影響を整理したいとき

## 関連ファイル

| 種別 | パス | 説明 |
|---|---|---|
| 実装 | 未確認 | 実ディレクトリ構成は要確認 |
| テスト | 未確認 | テスト配置ルールは要確認 |
| 関連docs | [docs/architecture/system-overview.md](./system-overview.md) | 全体像の確認 |

## 基本方針

- 配置ルールは責務で分け、技術詳細で分けすぎない
- 既存構造がある場合は新規追加より整合性を優先する
- 未使用ディレクトリや一時ファイルを長期文書に前提化しない

## 実装ルール

- アプリ本体の配置: 要確認
- 共通部品の配置: 要確認
- 設定ファイルの配置: 要確認
- テストコードの配置: 要確認
- ドキュメント更新時は実態と差分がないか確認する

## よくある変更パターン

### パターン1: 新規ディレクトリを追加する

- 手順:
  1. 既存の責務分割で吸収できないか確認する
  2. 長期的な保守単位として妥当か確認する

### パターン2: ファイルを移動する

- 手順:
  1. import や参照への影響を確認する
  2. 関連 `docs` のパス表記を更新する
