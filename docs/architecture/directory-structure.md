# Directory Structure

## AI向け要約

- この文書は、現在の Next.js プロジェクト構成と責務を整理します。
- 実装本体は `src/app`、静的資産は `public`、旧 HTML 試作は `HTML/` にあります。
- 新規ファイルの配置先を判断するときに参照します。

## このドキュメントを読むべきとき

- 新しいファイルの配置先を決めるとき
- 命名規則や責務分離を確認したいとき
- ディレクトリ再編の影響を整理したいとき

## 関連ファイル

| 種別 | パス | 説明 |
|---|---|---|
| 実装 | `src/app` | App Router 配下の画面・レイアウト・グローバル CSS |
| テスト | `tests` | `node:test` による最小スモークテスト |
| 静的資産 | `public` | Next.js が配信する公開ファイル |
| 旧試作 | `HTML` | 比較用の静的プロトタイプ |
| 関連docs | [docs/architecture/system-overview.md](./system-overview.md) | 全体像の確認 |

## 基本方針

- 配置ルールは責務で分け、技術詳細で分けすぎない
- 既存構造がある場合は新規追加より整合性を優先する
- 未使用ディレクトリや一時ファイルを長期文書に前提化しない

## 実装ルール

- アプリ本体の配置: `src/app`
- 共通部品の配置: 必要になったら `src/components` を追加する
- 設定ファイルの配置: ルート直下
- テストコードの配置: `tests`
- ドキュメント更新時は実態と差分がないか確認する

## よくある変更パターン

### パターン1: 新規ディレクトリを追加する

- 手順:
  1. 既存の責務分割で吸収できないか確認する
  2. 長期的な保守単位として妥当か確認する

### パターン2: ファイルを移動する

- 手順:
  1. import や参照への影響を確認する
  2. 関連 `docs` のパス表記を更新する

## 実行コマンド

```bash
make check
npm run dev
```
