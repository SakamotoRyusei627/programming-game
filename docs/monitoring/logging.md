# Monitoring Logging

## AI向け要約

- アプリケーションログの粒度、出力先、秘匿対象を整理する文書です。
- ログ基盤やフォーマットは未確認です。
- ログ追加や機微情報の見直し時に参照します。

## このドキュメントを読むべきとき

- 新しいログを出力するとき
- 障害調査に必要な情報が足りないとき
- 機微情報が出力されていないか確認したいとき

## 基本方針

- 調査に必要な情報を残しつつ、機微情報は出さない
- 例外時の相関情報を揃える
- ノイズになる定常ログを増やしすぎない

## 実装ルール

- ログレベル基準: 要確認
- 出力フォーマット: 要確認
- リクエスト相関ID有無: 要確認
- マスキング対象: 要確認
- 変更時は監視コストと検索性を確認する

## よくある変更パターン

### パターン1: 調査用ログ追加

- 手順:
  1. 何を切り分けたいログか明確にする
  2. 個人情報や秘密情報が出ないか確認する

### パターン2: ログ削減

- 手順:
  1. 運用で使っていないノイズを確認する
  2. 障害調査に必要な情報が残るか確認する
