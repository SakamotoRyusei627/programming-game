# Security HTML Sanitize

## AI向け要約

- HTML やリッチテキストを表示するときの安全化方針を整理する文書です。
- 現時点では表示実装やサニタイズライブラリは未確認です。
- ユーザー入力HTMLや外部HTMLを表示する変更時に参照します。

## このドキュメントを読むべきとき

- HTML 文字列を画面へ描画するとき
- リッチテキスト機能を追加するとき
- XSS リスクの確認が必要なとき

## 関連ファイル

| 種別 | パス | 説明 |
|---|---|---|
| 実装 | 未確認 | HTML表示実装は要確認 |
| テスト | 未確認 | XSS防御テストは要確認 |
| 関連docs | [docs/security/csp.md](./csp.md) | ブラウザ側防御 |

## 基本方針

- 信頼できないHTMLはそのまま描画しない
- サニタイズ責務とエスケープ責務を混同しない
- 許可リストを基本にする

## 実装ルール

- サニタイズライブラリ: 要確認
- 許可タグ/属性: 要確認
- 表示前後の変換処理: 要確認
- サーバ/クライアントどちらで実施するか: 要確認
- 変更時はテストケースも更新する

## よくある変更パターン

### パターン1: HTML 表示追加

- 手順:
  1. 入力元の信頼境界を確認する
  2. サニタイズ位置と許可範囲を決める

### パターン2: 許可属性追加

- 手順:
  1. XSS リスクを確認する
  2. 既存描画影響とテストを更新する
