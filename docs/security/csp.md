# Security CSP

## AI向け要約

- Content Security Policy の方針、例外、導入範囲を整理する文書です。
- 現時点ではヘッダ設定方式やポリシー内容は未確認です。
- スクリプト読み込みや外部リソース制御を変えるときに参照します。

## このドキュメントを読むべきとき

- CSP を追加・変更するとき
- 外部スクリプトや iframe を導入するとき
- ブラウザ上のセキュリティ制約を確認したいとき

## 基本方針

- まず厳しいポリシーを基準にし、必要な例外だけ追加する
- 例外は理由と対象を明確にする
- 開発用例外を本番前提にしない

## 実装ルール

- ヘッダ設定場所: 要確認
- nonce/hash 利用有無: 要確認
- report-only 運用有無: 要確認
- 外部ドメイン許可方針: 要確認
- 変更時は必要最小限の緩和にとどめる

## よくある変更パターン

### パターン1: 外部スクリプト追加

- 手順:
  1. 本当に必要か確認する
  2. 許可先と理由を文書化する

### パターン2: CSP エラー調査

- 手順:
  1. ブロック対象と実行文脈を特定する
  2. 例外追加か実装変更かを判断する
