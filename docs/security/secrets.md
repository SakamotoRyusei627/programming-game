# Security Secrets

## AI向け要約

- 秘密情報、認証情報、環境変数の管理ルールを整理する文書です。
- 実際の保管方法やツールは未確認です。
- 新しい認証情報追加や漏えい防止ルール整理時に参照します。

## このドキュメントを読むべきとき

- API キーや秘密鍵を追加するとき
- `.env` やシークレットストア運用を確認したいとき
- ログやドキュメントに値を書いていないか確認したいとき

## 関連ファイル

| 種別 | パス | 説明 |

| 実装 | 未確認 | シークレット参照実装は要確認 |

## 基本方針

- 値そのものは `docs` やコードへ書かない
- 環境ごとの差分は参照方法だけ記録する
- ローテーションや失効手順の有無を確認する

## 実装ルール

- 管理方式: 要確認
- `.env` 利用有無: 要確認
- ローカル開発の配布方法: 要確認
- ローテーション方針: 要確認
- 変更時はログとCI設定への露出を確認する

## よくある変更パターン

### パターン1: 新規シークレット追加

- 手順:
  1. 値の保管先と参照方法を確認する
  2. 全環境への反映要否を確認する

### パターン2: 漏えい疑い対応

- 手順:
  1. 露出範囲を切り分ける
  2. 失効、再発行、関連設定更新を整理する
