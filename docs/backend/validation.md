# Backend Validation

## AI向け要約

- サーバ側入力検証、ドメイン制約、エラー返却方針を整理する文書です。
- 検証ライブラリや実装場所は未確認です。
- 入力制約変更時に参照します。

- 新しい入力項目を追加するとき
- サーバ側制約を変更するとき
- フロントとの役割分担を確認したいとき

## 基本方針

- 最終的な制約保証はサーバ側で行う
- 技術的検証と業務的検証の境界を意識する
- エラーメッセージは利用者向けと内部向けを分ける

## 実装ルール

- アノテーション/コード検証方式: 要確認
- 例外変換: 要確認
- 共通バリデータの有無: 要確認
- 国際化対応の有無: 要確認
- 変更時は既存データと互換性を確認する

## よくある変更パターン

### パターン1: 項目制約追加

- 手順:
  1. 入力元すべてに影響があるか確認する
  2. エラー形式とテストを更新する

### パターン2: 業務制約追加

- 手順:
  1. 単純入力検証か業務ロジックかを切り分ける
  2. 実行位置と例外形式を整理する
