# Frontend Form Validation

## AI向け要約

- 入力フォームの検証方針、エラーメッセージ、サーバ側検証との役割分担を整理する文書です。
- 実装方式は未確認です。
- フォーム追加や入力制約変更のときに参照します。

## このドキュメントを読むべきとき

- フォームを新規追加するとき
- 入力制約やエラーメッセージを変えるとき
- フロントとバックの検証責務を確認したいとき

## 基本方針

- フロント側は即時フィードバックを担う
- 正式な制約判定はサーバ側とも整合させる
- 文言と表示位置は一貫性を保つ

## 実装ルール

- 検証ライブラリ: 要確認
- エラーメッセージ方針: 要確認
- 入力変換の責務: 要確認
- 送信前検証と入力中検証の使い分け: 要確認
- 変更時はアクセシビリティも確認する

## よくある変更パターン

### パターン1: 新規入力項目追加

- 手順:
  1. サーバ側制約と一致するか確認する
  2. エラー表示位置と文言を揃える

### パターン2: 検証ルール変更

- 手順:
  1. 既存データへの影響を確認する
  2. テストケースを更新する
