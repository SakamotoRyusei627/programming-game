# Frontend Components

## AI向け要約

- UIコンポーネントの責務分離、再利用方針、配置ルールを整理する文書です。
- 実装未確認のため、現時点では一般的な確認項目のみです。
- UI変更や共通部品追加のときに参照します。

## このドキュメントを読むべきとき

- 新しいコンポーネントを作るとき
- 既存コンポーネントの責務を見直すとき
- 共通化の判断をするとき

## 基本方針

- 表示責務とデータ取得責務を必要に応じて分離する
- 共通化は利用箇所と保守コストで判断する
- 既存命名・props設計があるならそれに合わせる

## 実装ルール

- 命名規則: 要確認
- 配置場所: 要確認
- Container / Presentational 分離の有無: 要確認
- 共通UI部品の管理方法: 要確認
- 変更時はアクセシビリティ影響も確認する

## よくある変更パターン

### パターン1: 共通コンポーネント追加

- 手順:
  1. 類似実装の有無を確認する
  2. props 設計を既存方針に合わせる

### パターン2: 既存画面のUI変更

- 手順:
  1. 影響範囲を画面単位で確認する
  2. テストとスタイル崩れ確認を行う
