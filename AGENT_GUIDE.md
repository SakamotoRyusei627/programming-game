# 共通エージェントガイド

このファイルは、このリポジトリで作業する AI コーディングエージェント向けの共通入口です。

`AGENTS.md` と `CLAUDE.md` は、このファイルを参照し、同じ指示に従うための薄い入口として扱います。

## 最初に読むもの

1. まず [docs/index.md](./docs/index.md) を読む
2. その後、作業内容に関係するカテゴリの `index.md` だけ読む
3. カテゴリ配下の詳細 `md` は必要になったものだけ読む
4. 実装指示がある場合は [TASKS.md](./TASKS.md) を確認する
5. 検討メモや要確認事項がある場合は [NOTES.md](./NOTES.md) を確認する
6. 過去の改善提案を確認したい場合は [PROPOSALS.md](./PROPOSALS.md) を確認する

## タスク実行ルール

- 実装指示は原則として [TASKS.md](./TASKS.md) の未完了タスクから受け取る
- タスク化前の検討、仕様の曖昧さ、比較案は [NOTES.md](./NOTES.md) に整理してよい
- 実装中に見つけた改善候補のうち、今回スコープ外で見送るものは [PROPOSALS.md](./PROPOSALS.md) に `提案` として残す
- 親タスクを確認したら、実装前に詳細タスクへ分解する
- 詳細タスクは `- [ ]` 形式で `TASKS.md` に追記し、進捗が追える状態にする
- `NOTES.md` の内容から、実装に必要なものだけを `TASKS.md` へ昇格させる
- リファクタリングで今回直せるものは `PROPOSALS.md` に逃がさず、今回変更内で対応する
- `PROPOSALS.md` には「何が問題か」「どう直すとよいか」「今回対応しない理由」を残す
- 実装は TDD で進める
- TDD の流れは `テスト実装 → テスト Red 確認 → 実装 → テスト Green 確認 → リファクタリング` を守る
- まず `make check` を実行し、変更前の状態を確認する
- 次にテストを実装し、対象テストが失敗することを確認する
- その後に必要最小限の実装を行い、対象テストが成功することを確認する
- Green 後にリファクタリングを行う
- リファクタリング後に再度テストを確認する
- 最後に再度 `make check` を実行し、問題がないことを確認する
- 実行コマンドは原則として [Makefile](./Makefile) のターゲットを使う
- `Makefile` のコマンドが未設定なら、推測で補わず対象プロジェクトに合わせて更新する

## 読み方ルール

- 関係ない `md` をまとめ読みしない
- 実装前に、作業対象に関係する `docs` だけ確認する
- 既存の命名規則・設計方針を確認してから実装する
- 不明点は推測で埋めず、`要確認` または `未確認` として扱う
- 推測で大規模リファクタしない
- `TASKS.md` にない大きな実装を独断で始めない

## docs 更新ルール

- 実装後、仕様・設計・コマンド・制約・注意点が変わった場合だけ `docs` を更新する
- 実装後に `docs` 更新が必要か判断する
- `docs` 配下の `index.md` 以外の文書は、`3,000文字以内を基本` とする
- `index.md` 以外の文書が `3,000文字` を大きく超えそうなら、別ファイルへの分割を検討する
- 分割した場合は、対応するカテゴリの `index.md` に新しい文書への導線を追加する
- 単なる作業ログや一時メモは `docs` に残さない
- 古くなった記述を見つけた場合は追記ではなく修正を優先する
- `md` を作成・更新するときは [docs/_templates/doc-template.md](./docs/_templates/doc-template.md) を参照する
- 過去の技術判断を追加するときは [docs/_templates/decision-template.md](./docs/_templates/decision-template.md) を参照する
- 実装フローや共通コマンドが変わった場合は `AGENT_GUIDE.md`、`TASKS.md`、`Makefile` の更新要否も確認する
- 検討メモ運用が変わった場合は `NOTES.md` の更新要否も確認する
- 改善提案の運用が変わった場合は `PROPOSALS.md` の更新要否も確認する

## 補足

- Claude Code を含む各エージェントは、必要な `docs` だけ読む
- 一時的な作業ログは `docs` に残さない
- エージェントごとの差分指示が必要になった場合だけ、各入口ファイルへ最小限の追記を行う
